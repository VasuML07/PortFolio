# StatVault AI – Machine Learning Subsystem

> Production-ready ML subsystem powering football intelligence with predictive analytics and player profiling.

## 🎯 Overview

- **Role:** AI & ML Developer
- **Timeline:** June 2026 - July 2026
- **Team:** Contributed to larger StatVault platform
- **Client:** StatVault AI Platform

## 🎯 The Challenge

Football analytics requires processing massive datasets to extract meaningful insights for match prediction and player evaluation. The challenge was to build a scalable ML subsystem capable of handling over 5 million player statistics and 245,000+ match records to provide accurate predictions and actionable player archetypes.

## 💡 Solution

Developed a comprehensive machine learning pipeline featuring an XGBoost model for match outcome prediction and unsupervised learning techniques for player profiling. Engineered complex features from raw data including historical form, ELO ratings, and expected goals (xG) to drive model performance.

## 🛠️ Technologies Used

- **Languages:** Python
- **ML Libraries:** Scikit-learn, XGBoost, NumPy, Pandas
- **Techniques:** Feature Engineering, K-Means Clustering, Cosine Similarity, Isolation Forest
- **Tools:** Git, Jupyter Notebook

## 📊 Results & Impact

- **Match Prediction:** Achieved 67.7% Accuracy and 0.811 ROC-AUC using XGBoost.
- **Player Analytics:** Successfully clustered players into 6 distinct archetypes (Poacher, Playmaker, Winger, etc.) using K-Means.
- **Data Scale:** Engineered features from a dataset containing 5,215,834 player statistics and 245,033 matches.
- **Anomaly Detection:** Implemented Isolation Forest to identify 50+ performance anomalies in player data.
